Items to request

  • Confirm which url’s we’ll be analyzing
  • Define audience
  • Determine goals

Access to request

This can vary from one client to the next depending upon the tools they utilize. This example is based off of a configuration which consists of the following:

  • WordPress
  • Google Analytics

WordPress access - Why?

We prefer to be granted admin rights to the client’s WordPress site.

  • Not having admin access could limit our ability to conduct a full analysis on the following audit items:
    • Index
    • Site verification
    • Redirects

WordPress access - How?

In order to set up CH users on the client’s WordPress site:

  • Provide email addresses of the developers working on the audit to the client
  • The client will need to add the new CH users per this documenation

Google Analytics access - Why?

We prefer to be granted “Edit” and ”Read and Analyze” rights to the client’s Google Analytics account.

  • Not having this level of access could limit our ability to conduct a full analysis on the following audit items:
    • Keywords
    • Sitewide performance
    • Site verification

Google Analytics access - How?

In order to set up CH users on the client’s Google Analytics account:

  • Each developer will need to create a Google Account that references their CH email address
  • Provide email addresses of the developers working on the audit to the client
  • The client will need to add the new CH users per this documentation
