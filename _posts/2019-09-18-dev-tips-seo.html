Background

  • CH recently conducted a SEO audit for Massachusetts Health Connector
  • As a result we would like to share information that developers need upfront in order to conduct an analysis of the technical items within the audit

Items to request

  • Confirm which url’s we’ll be analyzing
  • Define audience
  • Determine goals

Access to request

This can vary from one client to the next depending upon the tools they utilize. This example is based off of MHC’s configuration which consists of the following:

  • WordPress
  • Google Analytics

WordPress access - Why?

We prefer to be granted admin rights to the client’s WordPress site.

  • Not having admin access could limit our ability to conduct a full analysis on the following audit items:
    • Index
    • Site verification
    • Redirects

WordPress access - How?

In order to set up CH users on the client’s WordPress site:

  • Provide email addresses of the developers working on the audit to the client
  • The client will need to add the new CH users via the below documentation:
    • https://make.wordpress.org/support/user-manual/customizing-wordpress/managing-users-on-your-wordpress-site/adding-users-to-your-wordpress-site/

Google Analytics access - Why?

We prefer to be granted “Edit” and ”Read and Analyze” rights to the client’s Google Analytics.

  • Not having this level of access could limit our ability to conduct a full analysis on the following audit items:
    • Keywords
    • Sitewide performance
    • Site verification

Google Analytics access - How?

In order to set up CH users on the client’s Google Analytics:

  • Each developer will need to create a Google Account that references their CH email address
  • Provide email addresses of the developers working on the audit to the client
  • The client will need to add the new CH users via the below documentation:
    • https://support.google.com/analytics/answer/1009702?hl=en
